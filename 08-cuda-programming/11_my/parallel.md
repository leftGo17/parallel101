# 并行计算深度对比：CPU vs. GPU vs. NPU

本文以向量加法任务（$C = A + B$）为切入点，探讨不同处理器架构在并行管理、内存控制和任务分发上的本质区别。

---

## 一、 多核 CPU：紧密调度
* SMT(两个线程竞争一个核心),
* SIMD(单核心，单指令多数据并行),
* MIMD(多核心，多指令多数据)

多核CPU 处理并行的方式是 **MIMD（多指令多数据）**，每个核心都是一个独立。。

### 1.1 并行实现：软件驱动的任务拆分
*   **任务分发**：硬件不会自动拆分循环。程序员必须通过软件（如 `std::thread`、OpenMP）手动将向量拆段。例如：核 0 处理 `0~999`，核 1 处理 `1000~1999`。
*   **指令级并行 (SIMD)**：在单个核心内部，通过 **AVX/SSE** 向量指令实现“小规模并行”。一条指令同时操作 8 或 16 个浮点数。这通常需要编译器或程序员显式调用向量原语。

### 1.2 内存管理：硬件自动驾驶
*   **自动预取 (Prefetching)**：硬件预取器会自动识别内存访问模式（如线性扫描 A、B 数组），提前将数据抓取进私有的 L1/L2 Cache。
*   **一致性协议 (MESI)**：如果核 0 修改了 C 的一部分，核 1 想要读取，硬件协议会自动处理缓存失效和同步，程序员无需介入。

> **特点**：内存管理极度省心，但任务分发需要软件层面显式编排。

## 二、 GPU：海量线程掩盖延迟 (SIMT)

GPU 采用 **SIMT（单指令多线程）**，通过成千上万个轻量级线程来抵消显存访问的高延迟。

### 2.1 并行实现：硬件驱动的任务分发
*   **任务分发**：程序员只需描述一个线程的计算逻辑（Kernel）。硬件调度器（Warp Scheduler）会自动将数万个线程分发到不同的 SM（流式多处理器）上。
*   **执行模式**：32 个线程组成一个 **Warp**，共用一个指令发射器。虽然线程多，但步调高度一致。

### 2.2 内存管理：硬件掩盖延迟
*   **内存带宽**：拥有极高的显存带宽（HBM/GDDR），但延迟很大。
*   **延迟掩盖**：GPU 不像 CPU 那样追求快速读写。当一个 Warp 等待显存数据时，硬件会瞬间切换到另一个就绪的 Warp 运行。
*   **Cache**：虽然有硬件 Cache，但作用更多是减少重复访问，而不是像 CPU 那样靠 Cache 掩盖主存延迟。

> **特点**：任务分发极度省心（全自动化），靠“人海战术”解决等待数据的尴尬。

---

## 三、 NPU：程序复制到多个node (SPMD)

NPU（如 TPU、昇腾）通常针对张量/矩阵运算设计，追求极致的能效比。

### 3.1 并行实现：编译器驱动的静态编排
*   **静态调度**：NPU 砍掉了昂贵的硬件调度电路。什么时候算、什么时候搬，全部由**编译器**在离线阶段定死。
*   **计算单元**：通常采用**脉动阵列 (Systolic Array)**。数据像波浪一样在计算单元间传递，不需要反复读写寄存器。

### 3.2 内存管理：软件显式控制
*   **Scratchpad Memory**：NPU 没有复杂的硬件 Cache 自动层级，而是提供由程序员/编译器可见的 **SRAM（片上高速缓存）**。
*   **显式搬运**：必须写显式的 **DMA（直接内存访问）** 指令，精确控制数据在 DRAM（内存）和 SRAM 之间的流动时机。如果搬运指令写错，计算单元会拿到错误的数据或空转。

> **特点**：为了省电和高性能，把所有的复杂度都甩给了软件（驱动和编译器）。

## 四、 核心差异总结 (向量 A + B 场景)

| 特性             | 多核 CPU（半自动化）      | GPU（自动化）              | NPU（手动管理）              |
| :--------------- | :------------------------ | :------------------------- | :--------------------------- |
| **任务谁来拆？** | **程序员/库 (软件)**      | **调度器 (硬件原语)**      | **编译器 (离线静态)**        |
| **数据谁来搬？** | **预取器 (硬件自动)**     | **硬件自动 + 线程切换**    | **程序员/编译器 (显式 DMA)** |
| **同步谁来管？** | **一致性协议 (硬件电路)** | **同步屏障 (原语)**        | **MPI通信协议**              |
| **计算核心**     | 几个强力 ALU (复杂逻辑)   | 数千个简易 ALU (SIMT)      | 巨大的矩阵引擎 (脉动阵列)    |
| **哲学**         | 掩盖延迟 (Low Latency)    | 提高吞吐 (High Throughput) | 确定性计算 (Determinism)     |
